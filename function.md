# 发电机租赁平台系统开发与实现功能清单
## 4.1 登录与权限模块实现
### 4.1.1 Spring Security 与JWT集成实现
系统采用Spring Security作为安全框架，结合JWT实现无状态认证；用户登录成功后，后端生成JWT令牌返回给前端，前端在后续请求的Authorization头中携带该令牌完成身份验证。

### 4.1.2 多角色权限控制实现
基于RBAC权限模型，通过`@PreAuthorize`注解在Controller方法层实现细粒度权限控制；为租户、商家、管理员三类角色分配不同权限集合，严格区分各角色可访问的功能与接口，防止越权操作。

### 4.1.3 用户登录/注册功能实现
1. **注册**：用户可完成账号注册流程，填写信息后发送Post请求，后端校验通过后将信息录入数据库，作为后续登录校验依据；
2. **登录权限区分**：普通用户登录后可进行加购设备、申请商家、下单等操作；账号申请商家身份后，登录同时拥有普通用户+商家的订单签名、设备上下架等权限；管理员账号仅能通过后台赋予，无法注册或申请；
3. **角色权限隔离**：不同角色登录后对应不同操作界面和功能权限，权限不互通。

## 4.2 前台系统功能实现
### 4.2.1 设备浏览与搜索功能
为租户提供发电机设备多维度浏览与搜索能力，具体支持：
- 关键字模糊匹配搜索；
- 功率/燃料类型/价格等条件精准筛选；
- 地理位置就近搜索；
- 结合用户行为的智能推荐；
- 列表、网格等多种展示视图切换。

### 4.2.2 设备详情查看功能
展示发电机设备完整信息，为租户租赁决策提供全面依据，包含：
- 技术参数、租赁价格（日/周/月）、押金、库存状态；
- 商家信息及信用评级；
- 设备实拍图查看；
- 商家咨询入口。

### 4.2.3 租赁下单功能
租户选定设备后的下单流程，核心功能：
- 选择租赁时长、配送方式（商家配送/自提）；
- 填写交付地点、联系人、联系电话等信息；
- 系统自动计算租金、押金、平台服务费等费用明细；
- 提交信息后生成租赁订单，并跳转至支付页面。

### 4.2.4 订单处理功能
实现租赁订单全流程状态跟踪与处理，核心能力：
- 覆盖订单待确认、待支付、配送中、使用中、待归还等全状态流转；
- 租户端：查看订单进度、了解状态变更；
- 商家端：处理订单确认、发货等操作；
- 状态变更时同步触发对应的消息通知，告知双方。

### 4.2.5 在线支付功能
集成主流第三方支付方式，实现便捷、安全的在线支付，核心功能：
- 集成支付宝等主流支付方式，生成统一收银台；
- 用户选择支付方式后，系统调用支付网关接口完成支付；
- 支付结果由支付平台回调同步至系统；
- 支付成功后，自动更新订单支付状态，并发送支付成功通知。

### 4.2.6 订单管理功能
为租户提供个人订单一站式管理入口，核心功能：
- 查看所有订单的详细信息（订单号、设备、金额、状态、交付信息等）；
- 按订单状态筛选订单；
- 跟踪订单实时进度；
- 支持订单取消、申请归还等操作。

### 4.2.7 评价与投诉功能
为用户提供租赁全流程的评价与投诉渠道，保障双方权益，核心功能：
- 对已完成的租赁订单进行服务评分、填写评价内容；
- 提交投诉建议，可上传相关凭证；
- 实时查看评价/投诉的处理进度与反馈结果；
- 评价结果作为商家信用评级的重要依据。

## 4.3 后台系统功能实现
### 4.3.1 客户管理功能
为商家提供专属的客户管理模块，核心功能：
- 查看合作租户的租赁历史记录；
- 查看租户的信用记录、评价反馈等信息；
- 实现客户信息的统一管理与维护，助力商家优化服务策略。

### 4.3.2 商家入驻功能
提供个人/企业商户的入驻申请与审核流程，核心功能：
- 入驻申请入口：申请人填写商户基础信息、选择服务类型；
- 凭证上传：需上传身份证（正面/反面）、营业执照、发电机权属证明（购机发票/设备登记证）、发电机实拍图等核心凭证；
- 协议确认：申请人阅读并同意《平台商户入驻协议》和《设备租赁安全承诺书》；
- 审核流程：提交申请后进入平台管理员的审核流程，审核通过后成为平台商家。

### 4.3.3 设备管理功能
为商家提供发电机设备全生命周期管理能力，核心功能：
- 发布新设备：填写设备参数、上传图片、设置价格/押金/配送规则；
- 设备信息编辑：修改已发布设备的各项信息；
- 设备上下架操作：根据库存/运营需求调整设备展示状态；
- 库存状态查看：实时了解设备可租/已租/维护状态；
- 支持批量操作与模板化管理，提升设备管理效率。

### 4.3.4 用户管理功能
为平台管理员提供全平台用户精细化管理入口，核心功能：
- 查看所有租户、商家、管理员的账号信息（用户名、用户ID、角色、手机号、注册时间、状态等）；
- 支持按条件搜索、筛选用户；
- 可进行账号状态修改（启用/禁用）、权限分配操作；
- 实现用户账号的统一管理与维护。

### 4.3.5 设备审核功能
管理员对商家发布的发电机设备进行合规性与真实性审核，核心功能：
- 查看商家提交的设备参数、实拍图、权属证明等信息；
- 审核通过：设备方可在前台展示，对用户可见；
- 审核驳回：填写驳回理由，并将结果通知商家；
- 保障平台设备信息的规范性、真实性。

### 4.3.6 平台营运功能
为管理员提供平台运营参数配置与日常运营管理能力，核心功能：
- 系统配置管理：设置平台服务费比例、押金退还天数、纠纷处理规则等；
- 系统通知发送：向平台用户发布公告、通知等信息；
- 首页内容管理：调整平台首页展示内容、推荐设备等；
- 实现平台日常运营的灵活配置与管理。

### 4.3.7 数据分析功能
整合平台全维度运营数据，为运营决策提供数据支撑，核心功能：
- 数据指标覆盖：交易金额、订单数量、用户增长、设备利用率、商家收益等；
- 数据可视化：生成折线图、饼图等可视化报表与图表；
- 时间维度筛选：支持按日/月/年筛选查看数据；
- 数据概览与详情联动，便于管理员掌握平台运营状态。

## 4.4 本章小结核心实现要点
各模块实现均遵循MVC架构原则，代码结构清晰，业务逻辑完整；实现了从用户认证、设备管理、租赁交易到后台运营的全流程功能，为系统稳定运行提供保障。
