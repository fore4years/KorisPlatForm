# 多场景文件上传功能开发计划

根据您的需求，我将为平台实现一套标准化的文件上传系统，支持商品图片、用户头像、商家资质等多场景，并包含安全校验。

## 1. 后端开发 (Spring Boot)

### 1.1 基础设施
- **配置**: 在 `application.yml` 中添加文件存储路径配置 (`file.upload-dir`)。
- **静态资源映射**: 配置 Spring MVC 将 `/uploads/**` 路径映射到本地磁盘目录，确保上传的文件可被访问。

### 1.2 接口开发 (`FileController`)
- **上传接口**: `POST /api/files/upload`
  - **参数**: `file` (MultipartFile), `type` (业务类型: avatar/product/cert)
  - **逻辑**: 
    - 校验文件类型 (JPG/PNG/PDF) 和大小 (Avatar<2MB, Others<5MB)。
    - 根据 `type` 将文件存入不同子目录 (如 `uploads/avatar/`)。
    - 重命名文件 (UUID) 防止冲突。
    - 返回可访问的 HTTP URL。

## 2. 前端开发 (Vue3 + Element Plus)

### 2.1 通用组件封装
- **`UploadImage.vue`**: 封装 `el-upload` 组件。
  - **功能**: 图片预览、上传进度、自动上传到后端、回显 URL。
  - **参数**: `v-model` (绑定 URL), `type` (业务类型), `limit` (大小限制)。
  - **场景**: 用于头像、商品图、证件照。

### 2.2 页面集成
- **商户申请页 (`MerchantApply.vue`)**: 
  - 将原本的 Base64 本地预览逻辑替换为真实的上传组件。
  - 支持身份证、营业执照、发电机实拍图的上传。
- **商户后台 (`MerchantHome.vue`)**:
  - 在发布/编辑设备弹窗中，将“图片URL”输入框替换为上传组件。
- **个人中心 (`UserProfile.vue`)**:
  - 实现真实的头像上传功能，替换现有的模拟按钮。

## 3. 安全与优化
- **后端安全**: 严格限制允许上传的文件扩展名和 MIME 类型，防止恶意文件上传。
- **前端体验**: 添加上传前的格式和大小预校验，提供友好的错误提示。

## 4. 验证计划
- 启动应用后，分别测试头像、营业执照、商品图的上传。
- 验证上传后的文件能否在页面正常显示。
- 验证不符合规则的文件（如超大文件）是否被正确拦截。
