# 电子合同与交付流程实现计划

本计划旨在实现租赁流程中的电子合同签署、商家订单确认与交付、以及交付验收功能。

## 1. 数据库变更
*   **Contracts 表**: 现有的 `contracts` 表结构已包含 `tenant_signed`, `merchant_signed`, `content`。
    *   *新增*: `pdf_url` (存储生成的合同PDF链接, 可选), `sign_date` (签署时间)。
*   **Orders 表**:
    *   *新增*: `delivery_type` (DELIVERY/PICKUP - 商家配送/自提), `delivery_evidence_url` (交付验收照片), `delivery_status` (PENDING/DELIVERING/DELIVERED/RECEIVED)。
    *   *注意*: 原有 `status` 字段已包含 `DELIVERED`, `RENTING` 等，可复用或精细化。为保持简单，我们在 `status` 字段中流转，辅助以 `delivery_evidence_url`。

## 2. 后端开发 (Spring Boot)
*   **ContractService**:
    *   `generateContract(Long orderId)`: 根据订单信息生成标准合同文本（HTML模板填充）。
    *   `signContract(Long contractId, Long userId, String role)`: 用户/商家签署合同（更新状态）。
    *   `getContract(Long orderId)`: 获取合同内容及签署状态。
*   **OrderService (更新)**:
    *   `confirmOrder(Long orderId)`: 商家确认订单，触发合同生成。
    *   `updateDelivery(Long orderId, String deliveryType)`: 更新交付方式。
    *   `uploadDeliveryEvidence(Long orderId, String evidenceUrl)`: 上传验收照片，将订单状态流转为 `RENTING` (租赁中)。
*   **Controller**:
    *   `ContractController`: 处理合同查看与签署。
    *   `OrderController`: 新增确认、交付、上传凭证接口。

## 3. 前端开发 (Vue 3)
*   **订单详情页 (`OrderDetail.vue`)**:
    *   **状态栏**: 清晰展示当前流转状态（待确认 -> 待签署 -> 待交付 -> 租赁中）。
    *   **合同模块**:
        *   显示“查看合同”按钮。
        *   弹窗展示合同内容。
        *   “签署”按钮（租户/商家各自签署）。
    *   **商家操作区** (仅商家可见):
        *   “确认订单”按钮。
        *   “发货/交付”设置（选择配送方式）。
    *   **交付验收区**:
        *   上传组件：支持上传验收照片。
        *   “确认收货/交付完成”按钮。
*   **合同组件 (`ContractModal.vue`)**: 封装合同展示与签署逻辑。

## 4. 验证计划
*   **流程测试**:
    1.  商家登录，确认新订单 -> 系统自动生成合同。
    2.  租户登录，查看订单 -> 签署合同。
    3.  商家登录，签署合同 -> 设置交付方式 -> 上传交付照片 -> 确认交付。
    4.  检查订单状态变更为 `RENTING`。
